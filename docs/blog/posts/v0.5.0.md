# grok_switch v0.5.0：Grok Build AI 对话、富文本渲染与手机续聊

v0.5.0 在原有供应商切换与账号池之上加入完整的 Grok Build AI 对话工作台。现在可以直接在 grok_switch 中开始或继续 Grok 会话，选择项目工作目录，查看当前模型与工具活动，并在同一局域网内通过手机安全配对后继续对话。聊天界面采用浅色 Obsidian 风格，并完整支持 Markdown、代码高亮、Mermaid 和数学公式。

<!-- more -->

## 在 grok_switch 中直接操作 Grok Build

首页“添加供应商”左侧新增“聊天”入口。grok_switch 通过 ACP 标准输入输出协议连接本机的 `grok agent stdio`，回复、思考过程、工具调用和权限请求会实时显示在页面中，不需要把 Grok Agent 的独立端口暴露给浏览器。

聊天功能要求电脑已经安装并登录 Grok Build，并且终端中可以正常执行：

```powershell
grok --version
```

工具调用默认需要确认，也可以在可信环境中为当前 Agent 开启自动批准。命令执行、项目文件读取和修改始终发生在运行 grok_switch 的电脑上。

## 历史会话可以搜索并继续

左侧对话库会读取 Grok 本地保存的历史会话，可以按标题、工作目录或模型搜索。选择旧会话后，grok_switch 会通过真实的 ACP `session/load` 恢复上下文，而不是只把历史消息重新显示一遍，因此可以接着之前的任务继续讨论和执行。

每个会话会展示当前模型、会话 ID 和工作目录。新会话可以使用默认目录，也可以由用户输入任意有效项目目录；该选择会被记住。历史回复会保留各自使用的模型，当前模型同时显示在顶部和右侧会话信息栏。

## AI Native 富文本回复

回复区域使用本地固定版本的渲染依赖，不依赖 CDN，支持：

- GFM 标题、列表、任务列表、表格、加粗、删除线、链接与引用；
- 多语言代码块自动高亮、语言标识和一键复制；
- 行内与块级 KaTeX / LaTeX 数学公式；
- 本地图片与网络图片；
- 搜索结果中的可点击数字引用；
- Mermaid 流程图、时序图、甘特图和思维导图。

Mermaid 使用严格安全模式渲染。本版本同时修复了流程图和思维导图中部分节点只显示形状、不显示文字的问题。

## 浅色 Obsidian 风格工作台

聊天页默认使用浅色三栏布局：左侧是对话库，中间是消息与输入区，右侧是模型、目录、会话 ID 和工具活动。左右侧栏可以独立隐藏，也可以像 Obsidian 一样拖动分隔线调整宽度。

侧栏宽度和折叠状态会保存在浏览器本地。双击分隔线可以恢复默认宽度，键盘方向键也能进行微调。窄屏设备会自动切换为抽屉交互，避免侧栏挤压消息内容。

## 手机扫码后管理供应商并继续对话

局域网访问默认关闭。电脑端在“设置”中开启“允许同一局域网的手机访问”后，服务会从仅监听 `127.0.0.1` 切换为监听 `0.0.0.0`，并生成一次性配对二维码。

手机和电脑连接同一个 Wi-Fi 或热点时，扫描二维码即可打开同一套页面。配对码有效期为 10 分钟且只能使用一次；配对成功后使用 HttpOnly、SameSite 会话 Cookie 进行认证。关闭局域网访问会立即撤销已经配对的手机会话。

局域网模式复用 grok_switch 自身的鉴权和 WebSocket Origin 检查，不会直接暴露 Grok Agent。仍然只建议在可信的家庭或办公网络中使用，并在 Windows 防火墙中仅允许专用网络；不要将端口映射到公网。

## 单 EXE 与本地资源

聊天网关、页面、Markdown 渲染器、Mermaid、KaTeX、代码高亮器和字体都内嵌在同一个 `grok_switch.exe` 中。目标电脑无需安装 Node.js，也不需要联网下载前端依赖；运行时只需本机已有 Grok Build。

本版本继续提供：

- `grok_switch.exe`
- `grok_switch.exe.sha256`

如果 Release 显示为未签名构建，首次运行时 Windows 可能出现 SmartScreen 提示。可以先使用 SHA-256 文件核对下载内容。

## 升级提示

升级前请先退出旧版托盘进程，再用 v0.5.0 的 `grok_switch.exe` 替换旧文件。供应商、账号池、设置、历史备份和 Grok 会话仍保存在用户目录中，不需要重新导入。

首次使用手机访问时需要重新开启局域网选项并扫描配对二维码。首次使用聊天时，请先在终端确认 `grok --version` 可用，再在聊天页选择项目工作目录并启动 Agent。
